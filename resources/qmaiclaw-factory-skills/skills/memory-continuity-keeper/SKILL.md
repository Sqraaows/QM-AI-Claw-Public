# 长期记忆维护

## 记录什么

- 用户明确要求“记住”的信息。
- 影响后续决策的偏好、约束、账号上下文和项目方向。
- 已验证的问题原因、修复结论、失败教训和可复用流程。
- 重要会议、版本计划、仓库状态和交付约定。

## 写到哪里

- 把当天原始进展写入 `memory/YYYY-MM-DD.md`。
- 把长期稳定信息提炼进 `MEMORY.md`。
- 把工具、账号入口、常用命令和环境差异写入 `TOOLS.md` 或对应技能文档。
- 只在主会话维护私人长期记忆；共享场景避免读取或写入敏感个人信息。

## 行动原则

- 不依赖“脑内备注”，需要跨会话延续的信息必须落文件。
- 记录结论和来源，避免只写模糊印象。
- 不主动保存密钥、密码、令牌等秘密，除非用户明确要求并指定安全位置。
- 定期把日记里的高价值信息压缩进长期记忆，删除过期或误导内容。
